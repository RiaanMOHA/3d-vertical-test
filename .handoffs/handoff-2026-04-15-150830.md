## Architecture and design notes

### Scene registry pattern

The file uses a **single THREE.Scene** with multiple **THREE.Group** children — one group per "scene" (building / interior). Groups toggle via `.visible`. This is simpler than juggling multiple `THREE.Scene` objects: camera, renderer, shadow map, and controls are all shared.

```js
const scenes = {};
let activeScene = null;

function registerScene(id, cfg) {
  const group = new THREE.Group();
  group.visible = false;
  scene.add(group);
  scenes[id] = {
    id, group,
    walls: [],          // per-scene collision meshes
    rooms: cfg.rooms,   // AABB list for label detection
    orbitTarget: cfg.orbitTarget,
    orbitPos:    cfg.orbitPos,
    fpStart:     cfg.fpStart,
    fpYaw:       cfg.fpYaw || 0,
    build:       cfg.build,
  };
  cfg.build(scenes[id]);   // build() receives S so it can push into S.walls
}
```

Each scene's `build(S)` function pushes collision walls into `S.walls`, adds meshes/lights to `S.group`, and defines `S.rooms` for label detection. `switchScene(id)` swaps group visibility, reframes the orbit camera, updates the scene background, and exits pointer lock if active.

### Wall-segment pattern (no CSG)

Every wall with an opening is split into 2–4 separate `BoxGeometry` pieces around the opening: sill below, header above, solid flanks. `addWall(x1, z1, x2, z2, y1, y2, exterior)` creates one segment at a time. This keeps geometry cheap and avoids three-csg-ts or similar.

### Per-axis FP collision for wall-sliding

```js
tryMove(new THREE.Vector3(m.x, 0, 0));   // attempt X only
tryMove(new THREE.Vector3(0, 0, m.z));   // then Z only
```
If one axis is blocked, the other still moves — the player slides along a wall instead of sticking.

### Procedural plank texture helper

`plankTex(baseHex, plankCount, seed)` returns a `THREE.CanvasTexture` built from a 512×512 Canvas 2D:
1. Fill with base color.
2. For each plank row: fill with a slightly randomised tint, draw a dark gap line at the top, overlay 24 random subtle grain strokes.
3. Wrap as repeating texture with `anisotropy: 8`.

Used for both ceiling and floor in the interior scene, and for the hallway floor glimpse.

### Minimal inline OrbitControls

~55 lines. Supports drag-to-rotate (Spherical coords), shift+drag pan (along camera right/up), wheel zoom (radius clamped 1–40). Added a `setFrom(pos, target)` method so scene switching can reframe the camera without destroying the controls instance.

### Chip UI

Two `.chip` divs top-centre with `pointer-events: auto` (siblings have `pointer-events: none` so they don't block the canvas). Active chip gets `.chip.active` styling (white pill, dark text). Click listeners call `switchScene(data-scene)`.
